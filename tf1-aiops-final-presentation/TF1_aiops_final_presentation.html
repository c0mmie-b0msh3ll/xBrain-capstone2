02 — AI EngineCompute-first pipeline
Detector Ensemble

Metrics diagnosis is not one rule

The engine combines threshold checks, statistical drift, shape anomaly, and log evidence.

Threshold

Latency ≥ 1000ms, error/5xx ≥ 5, availability < 95, CPU/memory ≥ 85.

Rolling Z-score

Latest point compared to baseline; abs(z) ≥ 3 creates high-severity evidence.

EWMA drift

α = 0.35; catches drift when current deviates from smoothed expected value.

IsolationForest

contamination = 0.15, random_state = 7; catches latest-point shape anomaly.

Log anomaly layer

Error-like logs are counted by service using tokens: error, timeout, failed, refused, exhausted, down, deadline. Count/5.0 becomes a bounded log score.

02 — AI EngineDetector ensemble
Topology + Causal Hints

RCA ranking uses service graph evidence

We rank candidate services from metric/log evidence, dependency hints, recent deploys, and lag correlation.

Candidate score inputs

  • Detector score from metric/log anomaly evidence.
  • Dependency bump: impacted service dependency gets +0.35.
  • Recent deploy bump: deployed service gets +0.45.
  • Causal hint bump: lag-correlation score × 0.3.
  • If no evidence exists, alerted service becomes weak anchor: 0.25.
RCA ranking logic
score[service] += detector_score if dependency_of_impacted_service: score[dependency] += 0.35 if recent_deploy_on_service: score[service] += 0.45 if lag_correlation_abs >= 0.70: score[source] += correlation * 0.30 confidence = min(score / top_score, 1.0) return top_5_candidates
Important: lag correlation is explicitly “supporting evidence, not proof.” That gives us a defendable RCA story without overclaiming causality.

QA Judge + Action Catalog

Final answer is checked before handoff

The QA judge penalizes unsupported diagnosis; action catalog keeps recommendations constrained.

QA fail examples

Missing evidence, unsupported classification, unsupported root cause, invented service/signal, unsafe action.

Confidence penalty

Deterministic QA can apply -0.1; LLM QA can return confidence_delta from -0.2 to 0.

Action catalog

Recommendations are IDs from a curated catalog, not free-form remediation.

Defense line: “The model cannot just invent an action. Recommendations are grounded in evidence and selected from an action catalog; execution remains human-owned.”

Q&A Defense

Answers we should keep short

Why no auto-remediation?

Client explicitly scoped it out. AI diagnoses and suggests; human confirms and acts.

What does AgentCore add?

Bounded investigator runtime. CDO can invoke it cross-account when needed.

Why DynamoDB?

Audit trail and idempotency replay. Every decision can be traced.

Where are Slack/Jira?

CDO owns dispatcher and mutation. AI returns CDO-compatible payload.

What was the worker bug?

Worker missed bearer token with service auth enabled. Fixed to avoid 401 on deploy.

What if confidence is low?

Status becomes INVESTIGATE. The engine does not guess.
